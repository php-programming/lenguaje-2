# Sección 3: GitHub como Plataforma de Ingeniería de Software

Si Git es el motor de control de versiones de alto rendimiento, GitHub es el chasis, la carrocería y el sistema de navegación que lo convierten en un vehículo completo para la ingeniería de software colaborativa. Esta sección se centra en cómo GitHub extiende Git para facilitar el trabajo en equipo, la automatización y la gestión integral del ciclo de vida del software.

Analizaremos el flujo de trabajo colaborativo central (`Pull Request`) y exploraremos las capacidades de la plataforma que habilitan las prácticas modernas de DevOps y DevSecOps.

## Contenidos

1.  **[Más Allá del Repositorio Remoto](./3.1_Mas_alla_del_Remoto.md)**: Se presenta a GitHub no como un simple "disco duro en la nube" para Git, sino como una plataforma integrada que incluye gestión de proyectos, seguridad y automatización.
2.  **[El Flujo de Colaboración Central: Pull Requests](./3.2_Flujo_de_Colaboracion.md)**: Se detalla el mecanismo de `Pull Request` como el corazón del desarrollo colaborativo, enfocándose en su rol para la revisión de código (`Code Review`) y la integración segura de cambios.
3.  **[Introducción a CI/CD con GitHub Actions](./3.3_GitHub_Actions_y_CI_CD.md)**: Se introduce el concepto de Integración Continua (CI) y Despliegue Continuo (CD) y cómo GitHub Actions permite automatizar los flujos de trabajo de construcción, prueba y despliegue directamente desde el repositorio.
4.  **[Conceptos Avanzados: IaC y GitOps](./3.4_GitOps_e_IaC.md)**: Se exploran paradigmas modernos donde Git se convierte en la fuente única de verdad no solo para el código de la aplicación, sino también para la definición de la infraestructura (Infraestructura como Código) y la operación de sistemas (GitOps).
