# 7.5 Interfaces: El Contrato

## El "Qué" (El Concepto)

Una **Interface** es un contrato 100% abstracto. No es una clase, no se puede instanciar (`new`). Es una lista de métodos que una clase debe implementar obligatoriamente.

La interfaz define el **"qué"** (qué métodos deben existir), pero nunca el **"cómo"** (la lógica dentro de esos métodos).

## El "Por Qué" (Polimorfismo)

Propósito: Garantizar que diferentes clases, aunque no tengan relación de herencia, puedan ser tratadas de la misma manera. Esto permite escribir código flexible que no se "amarra" a una clase específica, sino a un comportamiento.

**Analogía:** Un enchufe de pared. La interfaz `iEnchufe` define un contrato: "debes tener dos patas planas". No le importa si eres un `CargadorDeTelefono` o una `Licuadora`. Si ambos `implements iEnchufe`, la pared sabrá cómo usarlos.

## Ejemplo Práctico

```php
<?php
// 1. EL CONTRATO (La Interfaz)
interface iAlmacenable {
    public function guardar($datos);
    public function leer($id);
}

// 2. IMPLEMENTACIÓN 1: Base de Datos
class AlmacenMySQL implements iAlmacenable {
    public function guardar($datos) { echo "INSERT INTO db... $datos"; }
    public function leer($id) { return "SELECT FROM db... $id"; }
}

// 3. IMPLEMENTACIÓN 2: Archivo de Texto
class AlmacenArchivo implements iAlmacenable {
    public function guardar($datos) { echo "fwrite()... $datos"; }
    public function leer($id) { return "fread()... $id"; }
}

// 4. EL CÓDIGO FLEXIBLE (Inyección de Dependencias)
// Esta función acepta CUALQUIER cosa que cumpla el contrato.
function procesar(iAlmacenable $repositorio) {
    $repositorio->guardar("Datos");
}
?>
```
