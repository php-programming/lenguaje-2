# 3.3 Guía de Conexión a Gestores de Bases de Datos

## Introducción

Independientemente de la herramienta GUI que elija, la conexión a un servidor de base de datos siempre requiere el mismo conjunto de parámetros. Comprender estos parámetros es fundamental.

### Parámetros de Conexión Universales

1.  **Hostname / Dirección del Servidor:** La dirección IP (ej. `127.0.0.1` para local) o el nombre de dominio (ej. `db.sisa.edu`) donde se está ejecutando el servidor de la base de datos.
2.  **Puerto (Port):** El "número de puerta" en el servidor que está escuchando las conexiones de la base de datos. Cada SGBD tiene un puerto por defecto:
    *   **MySQL:** `3306`
    *   **SQL Server:** `1433`
    *   **PostgreSQL:** `5432`
3.  **Nombre de Usuario (Username):** El usuario que ha sido creado en el SGBD para autenticarse.
4.  **Contraseña (Password):** La contraseña asociada a ese usuario.
5.  **Base de Datos por Defecto (Opcional):** El nombre de la base de datos específica a la que desea conectarse una vez autenticado.

## Guías de Conexión

### 1. MySQL Workbench

1.  **Abrir MySQL Workbench:** En la pantalla de bienvenida, haga clic en el icono `+` junto a "MySQL Connections".
2.  **Configurar Parámetros:**
    *   **Connection Name:** Un nombre descriptivo para su conexión (ej. "SISA_Local").
    *   **Hostname:** `127.0.0.1` (si su BD está en su misma máquina).
    *   **Port:** `3306` (generalmente no se cambia).
    *   **Username:** `root` o el usuario que haya creado.
3.  **Probar Conexión:** Haga clic en "Test Connection". Le pedirá la contraseña. Ingrésela. Debería ver un mensaje de "Successfully made the MySQL connection".
4.  **Guardar:** Haga clic en "OK" para guardar la conexión. Aparecerá en su pantalla de inicio para futuros accesos.

### 2. SQL Server Management Studio (SSMS)

1.  **Abrir SSMS:** El cuadro de diálogo "Connect to Server" aparecerá automáticamente.
2.  **Configurar Parámetros:**
    *   **Server type:** `Database Engine`.
    *   **Server name:** Puede ser `.` o `(localdb)\MSSQLLocalDB` para una instancia local, o el nombre del servidor.
    *   **Authentication:**
        *   **Windows Authentication:** (Por defecto) Usa su usuario de Windows si está configurado.
        *   **SQL Server Authentication:** Deberá proporcionar un `Login` (usuario) y `Password`.
3.  **Conectar:** Haga clic en "Connect". El "Object Explorer" a la izquierda se poblará con las bases de datos del servidor.

### 3. Beekeeper Studio

1.  **Abrir Beekeeper:** Haga clic en "Import from URL" o "New Connection".
2.  **Seleccionar Tipo de BD:** Elija el icono del SGBD al que se quiere conectar (ej. MySQL, SQL Server).
3.  **Configurar Parámetros:** Rellene los campos `Host`, `Port`, `User`, `Password` y `Database`. Beekeeper proporciona pestañas separadas para configuraciones avanzadas como SSL o túneles SSH.
4.  **Probar y Guardar:** Haga clic en "Test". Si la prueba es exitosa, haga clic en "Save". La conexión se guardará en el panel izquierdo.

### 4. phpMyAdmin

phpMyAdmin es una aplicación web, por lo que el proceso es diferente. No se "conecta" desde una app de escritorio.
1.  **Acceso:** Abra su navegador y vaya a la URL donde está instalado phpMyAdmin (ej. `http://localhost/phpmyadmin`).
2.  **Autenticación:** La página de inicio es una pantalla de login. Ingrese el `Usuario` y `Contraseña` de la base de datos.
3.  **Conexión:** Al iniciar sesión, ya está conectado. El servidor y la conexión (`localhost`) fueron pre-configurados cuando se instaló phpMyAdmin. Las bases de datos disponibles aparecerán en el panel izquierdo.

### 5. DBeaver (Cliente Universal)

1.  **Abrir DBeaver:** Haga clic en el icono de "New Database Connection" (un enchufe con un `+`).
2.  **Seleccionar Driver:** Aparecerá un asistente. Elija el SGBD al que se quiere conectar (ej. "MySQL"). DBeaver podría pedirle descargar el "driver" (un conector de software) la primera vez.
3.  **Configurar Parámetros:** En la pestaña "Main", rellene los campos `Server Host`, `Port`, `Database`, `Username` y `Password`.
4.  **Probar y Finalizar:** Haga clic en "Test Connection...". Si es exitoso, haga clic en "Finish". Su nueva conexión aparecerá en el panel "Database Navigator".

### 6. DataGrip (IDE de JetBrains)

1.  **Abrir DataGrip:** Vaya a `File > New > Data Source` y seleccione el SGBD (ej. "MySQL").
2.  **Configurar Parámetros:** En la parte inferior de la ventana, rellene los campos `Host`, `Port`, `User`, `Password` y `Database`.
3.  **Descargar Driver:** Si es necesario, DataGrip mostrará un enlace para descargar el driver JDBC requerido.
4.  **Probar y Aplicar:** Haga clic en "Test Connection". Si es exitoso, haga clic en "Apply" y luego en "OK". La fuente de datos aparecerá en el panel "Database Explorer".
