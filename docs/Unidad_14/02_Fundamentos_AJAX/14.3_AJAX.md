# 14.3 Definición de AJAX (Asynchronous JavaScript and XML)

## El Cambio de Paradigma

En la web clásica (1.0), cada clic recargaba toda la página. Esto es lento e ineficiente.
**AJAX** es la técnica que permite al navegador comunicarse con el servidor en segundo plano (asíncronamente) sin interferir con la visualización de la página actual.

*   **A**synchronous: No bloquea al usuario.
*   **J**avaScript: El motor que hace la petición.
*   **A**nd **X**ML: El formato de datos original (hoy en día, usamos **JSON** casi exclusivamente).

## La API `fetch` (El Estándar Moderno)

Olvidemos `XMLHttpRequest` (la forma antigua). Hoy usamos `fetch`.

```javascript
// Petición GET asíncrona
fetch('api/productos.php')
    .then(response => response.json()) // Convertir respuesta a JSON
    .then(data => {
        console.log(data); // Aquí tenemos los datos del servidor
        // Ahora podemos usar el DOM para mostrarlos
    })
    .catch(error => console.error('Error:', error));
```

**Flujo de Datos:**
1.  **JS (Cliente):** `fetch('api.php')` -> Envía petición HTTP.
2.  **PHP (Servidor):** Recibe petición -> Consulta BD -> `echo json_encode($datos)`.
3.  **JS (Cliente):** Recibe JSON -> Actualiza el DOM (HTML).
