# 14.1 El Modelo de Objeto de Documento (DOM)

## Caso de Uso

**Escenario:** Un formulario de registro donde, al seleccionar "País: Venezuela", el campo "Estado" debe cargarse automáticamente con los estados de Venezuela, sin recargar la página.
**Requisito:** Necesitamos manipular la estructura HTML de la página *después* de que el navegador la ha cargado.

---

## ¿Qué es el DOM?

El **DOM (Document Object Model)** es una representación en memoria (un árbol de objetos) de la estructura HTML de una página web.
*   Para el navegador, el HTML es solo texto.
*   El DOM es la "API" que el navegador ofrece a JavaScript para leer y modificar ese HTML.

### Manipulación con JavaScript

JavaScript es el único lenguaje que se ejecuta en el navegador (Cliente).

```javascript
// 1. Seleccionar un elemento (El Nodo)
const boton = document.getElementById('btn-guardar');
const inputNombre = document.querySelector('#nombre');

// 2. Escuchar Eventos (Interacción)
boton.addEventListener('click', function() {
    // 3. Manipular el DOM
    if (inputNombre.value === '') {
        inputNombre.style.border = '2px solid red'; // Cambiar estilo CSS
        alert('El nombre es obligatorio');
    } else {
        // Crear un nuevo elemento dinámicamente
        const mensaje = document.createElement('p');
        mensaje.textContent = 'Guardando...';
        document.body.appendChild(mensaje);
    }
});
```

**Importancia en la Arquitectura:**
El DOM es el puente entre el usuario y la lógica. Antes de enviar datos al servidor (Backend), usamos JS y el DOM para validar y mejorar la experiencia de usuario (UX).
