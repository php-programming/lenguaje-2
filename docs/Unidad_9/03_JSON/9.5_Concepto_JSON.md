# 9.5 JSON (JavaScript Object Notation)

Ahora que tenemos la estructura de nuestros modelos de datos en POO, es crucial entender cómo estos datos viajan y se comunican con el exterior (ej. una aplicación móvil, JavaScript en el navegador, o un servicio de terceros). Aquí es donde entra en juego JSON.

## Concepto Teórico: ¿Qué es JSON?

JSON es el formato estándar de intercambio de datos más popular en la web moderna. Su simplicidad, legibilidad y facilidad de parseo (análisis) en casi cualquier lenguaje de programación lo hacen indispensable.

- **Formato de Intercambio de Datos**: Es un formato de texto ligero diseñado para el intercambio de datos entre sistemas.
- **Basado en Objetos**: Se basa en la notación de objetos de JavaScript, pero es completamente independiente de cualquier lenguaje.
- **Estructura**: Está compuesto por pares de clave-valor y puede anidar arrays (listas) y objetos (diccionarios).

| Estructura PHP   | Equivalente JSON | Descripción                        |
| :--------------- | :--------------- | :--------------------------------- |
| Array Asociativo | Objeto (`{ }`)   | Un conjunto de datos clave: valor. |
| Array Indexado   | Array (`[ ]`)    | Una lista ordenada de valores.     |

**Ejemplo de Estructura JSON:**

```json
{
  "nombre": "Laptop de Prueba",
  "precio": 1200.0,
  "activo": true,
  "etiquetas": ["electronica", "computo", "oferta"],
  "vendedor": {
    "nombre": "Tech Corp",
    "pais": "MX"
  }
}
```
