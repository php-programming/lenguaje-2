# 9.1 El Patrón MVC (Modelo-Vista-Controlador)

## Concepto Teórico

Antes de hablar del Controlador de Modelos, es vital entender el patrón de diseño MVC, que separa las preocupaciones de una aplicación.

- **Modelo (Model)**: La capa de datos. Se encarga de la lógica de negocio, la conexión a la base de datos, y las operaciones de CRUD (Crear, Leer, Actualizar, Borrar). El modelo nunca interactúa directamente con el usuario.
- **Vista (View)**: La capa de presentación. Se encarga de mostrar la interfaz de usuario (HTML, CSS, JS) y los datos que le envía el Controlador. La Vista nunca contiene lógica de negocio.
- **Controlador (Controller)**: La capa de lógica de control. Actúa como el intermediario entre el Modelo y la Vista. Recibe las peticiones del usuario, decide qué Modelos deben ser invocados para obtener/modificar datos, y luego selecciona la Vista adecuada para mostrar el resultado.
