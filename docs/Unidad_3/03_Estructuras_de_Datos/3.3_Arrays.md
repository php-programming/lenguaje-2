# 3.3 Arrays: Estructuras de Datos en el Backend

## Caso de Uso

**Escenario:** Un carrito de compras en un e-commerce.
**Problema:** Un usuario selecciona 5 productos diferentes. Crear variables `$producto1`, `$producto2`, etc., es insostenible.
**Solución:** Utilizar una estructura de datos que pueda almacenar una colección dinámica de elementos bajo un mismo identificador.

---

## El Array en PHP

En PHP, el `array` es una estructura de datos extremadamente versátil. A diferencia de C o Java, donde existen arrays, listas enlazadas, mapas hash y colas como estructuras separadas, el array de PHP es en realidad un **mapa ordenado**. Puede comportarse como todas las anteriores.

### 1. Arrays Indexados (Vectores)

Son colecciones ordenadas donde las claves son números enteros, comenzando por defecto en 0.

```php
$tecnologias = ["PHP", "Java", "Python"];
// Acceso: $tecnologias[1] devuelve "Java"
```

### 2. Arrays Asociativos (Diccionarios / Hash Maps)

Son colecciones donde las claves son cadenas de texto personalizadas. Son fundamentales para representar entidades del mundo real (como un registro de base de datos).

```php
$usuario = [
    "id" => 101,
    "nombre" => "Carlos",
    "email" => "carlos@sisa.edu",
    "activo" => true
];
// Acceso: $usuario['email']
```

### 3. Arrays Multidimensionales

Arrays que contienen otros arrays. Es la estructura estándar para manejar conjuntos de registros (ej. el resultado de `SELECT * FROM productos`).

```php
$inventario = [
    ["producto" => "Laptop", "stock" => 10],
    ["producto" => "Mouse", "stock" => 50]
];
// Acceso: $inventario[0]['stock'] devuelve 10
```

### Manipulación y Depuración

*   **`print_r($array)`:** Imprime información legible para humanos sobre una variable. Ideal para ver la estructura de un array anidado.
*   **`count($array)`:** Devuelve el número de elementos.
*   **`array_push($array, $valor)`:** Añade un elemento al final (como una pila).

**Perspectiva de Ingeniería:**
El uso eficiente de arrays reduce la complejidad del código. En lugar de pasar 10 variables a una función, se pasa un solo array de configuración. Sin embargo, el abuso de arrays asociativos sin estructura definida ("array soup") puede llevar a código difícil de mantener. En sistemas grandes, se prefiere evolucionar de Arrays a **Objetos (Clases)** para garantizar la integridad de la estructura de datos (DTOs - Data Transfer Objects).
