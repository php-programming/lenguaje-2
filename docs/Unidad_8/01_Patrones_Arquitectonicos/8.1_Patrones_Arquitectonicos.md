# 8.1 Patrones Arquitectónicos: El Plano del Edificio

## El Problema de la Complejidad

Cuando una aplicación crece, se vuelve compleja. Sin un "plano" (arquitectura), terminamos con código inmanejable. Un **Patrón Arquitectónico** es una solución probada para organizar la estructura de una aplicación.

## Patrones Comunes

1.  **Capas (Layered):** Separa el código en estratos horizontales (Presentación, Lógica, Datos).
2.  **Cliente-Servidor:** Separa quien pide (Frontend) de quien sirve (Backend).
3.  **MVC (Modelo-Vista-Controlador):** El estándar de oro para la web.

### El Patrón MVC

*   **Modelo (M):** Datos y Lógica de Negocio. Habla con la BD.
*   **Vista (V):** Interfaz de Usuario (HTML). Muestra datos.
*   **Controlador (C):** Intermediario. Recibe la petición, coordina al Modelo y selecciona la Vista.

## El Enrutador (Router)

Es el "recepcionista" de la aplicación. Su único trabajo es mirar la URL (`/perfil/1`) y decidir qué Controlador debe atenderla (`UsuarioController::verPerfil(1)`).
