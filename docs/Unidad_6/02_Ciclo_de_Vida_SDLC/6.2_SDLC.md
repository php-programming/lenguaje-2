# 6.2 El Ciclo de Vida del Desarrollo de Sistemas (SDLC)

## Metodología de Ingeniería

El SDLC (Systems Development Life Cycle) es el marco que estructura el proceso de creación de software para garantizar calidad y eficiencia. Evita el "Code and Fix" (programar a lo loco y arreglar después).

### Fases del SDLC

1.  **Análisis de Requisitos (El "Qué"):**
    *   Hablar con los stakeholders.
    *   Definir Requisitos Funcionales (ej. "El sistema debe enviar un email") y No Funcionales (ej. "El sistema debe responder en <200ms").
    *   *Entregable:* Documento de Especificación de Requisitos.

2.  **Diseño (El "Cómo"):**
    *   Arquitectura del sistema.
    *   Diseño de Base de Datos (Diagramas ERD, Normalización - Unidad 2).
    *   Diseño de API.
    *   *Entregable:* Documentos de Diseño Técnico.

3.  **Implementación (Codificación):**
    *   Escribir el código siguiendo estándares (Unidades 3 y 4).
    *   Aplicar prácticas de seguridad (Unidad 5).
    *   Control de versiones (Unidad 1).

4.  **Pruebas (Testing):**
    *   Unit Testing, Integration Testing, User Acceptance Testing (UAT).
    *   Verificar que se cumplen los requisitos.

5.  **Despliegue y Mantenimiento:**
    *   Puesta en producción (CI/CD).
    *   Monitoreo y corrección de bugs.

### Modelos de Proceso

*   **Cascada (Waterfall):** Secuencial. Rígido. Bueno para sistemas críticos (ej. medicina, aeroespacial) donde los requisitos no cambian.
*   **Ágil (Agile/Scrum):** Iterativo e incremental. Flexible. El estándar en la industria web. Se entrega valor en ciclos cortos (Sprints).
