# 6.1 Roles en la Ingeniería de Software

## El Ecosistema de Especialización

El desarrollo de software moderno es demasiado complejo para que una sola persona domine todas las áreas a profundidad. Los equipos se organizan en roles especializados.

### 1. Frontend Engineer (UI/UX)
*   **Enfoque:** Lo que el usuario ve e interactúa.
*   **Tecnologías:** HTML5, CSS3, JavaScript (React, Vue, Angular).
*   **Responsabilidad:** Accesibilidad, rendimiento de renderizado, experiencia de usuario (UX).

### 2. Backend Engineer (Lógica y Datos)
*   **Enfoque:** Lo que sucede "detrás del telón".
*   **Tecnologías:** PHP, Python, Java, Go, Node.js, SQL.
*   **Responsabilidad:** APIs, lógica de negocio, seguridad, escalabilidad, integridad de datos.
*   *Conexión con el curso:* Todo lo visto en las Unidades 2, 3, 4 y 5 es el núcleo de este rol.

### 3. Full-Stack Engineer
*   **Enfoque:** Visión holística. Capaz de implementar una funcionalidad de punta a punta (End-to-End).
*   **Reto:** Mantenerse actualizado en ambos mundos es difícil. A menudo son "T-shaped" (profundos en uno, competentes en el otro).

### 4. DevOps / SRE (Site Reliability Engineering)
*   **Enfoque:** Infraestructura, automatización y operaciones.
*   **Tecnologías:** Docker, Kubernetes, AWS/Azure, CI/CD (GitHub Actions), Terraform.
*   **Responsabilidad:** Que el código pase del repositorio a producción de forma segura y que el sistema no se caiga.
*   *Conexión con el curso:* Unidad 1 (Git) y Unidad 3 (Servidores).

## El Rol del Ingeniero de Sistemas

Más allá de escribir código, el Ingeniero de Sistemas es un **arquitecto**. Su valor radica en:
1.  **Toma de Decisiones:** Elegir la base de datos correcta (SQL vs NoSQL) basada en el Teorema CAP.
2.  **Gestión de Riesgos:** Anticipar fallos de seguridad y cuellos de botella.
3.  **Visión Sistémica:** Entender cómo un cambio en el Frontend afecta la carga en la Base de Datos.
